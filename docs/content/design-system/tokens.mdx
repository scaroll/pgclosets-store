# Design Tokens

PG Closets uses a unified design token system that ensures consistency across all digital properties. These tokens are the single source of truth for all design decisions.

## Color System

### Brand Colors

Our primary brand palette reflects the premium, Apple-inspired aesthetic:

```tsx
import { colors } from '@/lib/design-tokens';

// Brand colors
colors.brand.black    // #0f1419
colors.brand.navy     // #243c74
colors.brand.sky      // #89bee6
```

### Gray Scale

Apple-inspired grayscale for premium feel:

```tsx
colors.gray[50]   // #ffffff - Lightest
colors.gray[100]  // #f5f5f7
colors.gray[500]  // #86868b - Mid
colors.gray[900]  // #1d1d1f - Darkest
```

### Material Palette

Industry-specific colors for closet and interior design:

```tsx
// Woodgrain tones
colors.materials.woodgrain.light   // #FAF8F6
colors.materials.woodgrain.DEFAULT // #8B7355
colors.materials.woodgrain.dark    // #2F261C

// Metal finishes
colors.materials.metal.roseGold    // #B76E79
colors.materials.metal.bronze      // #CD7F32
colors.materials.metal.champagne   // #F7E7CE
```

### Semantic Colors

Contextual colors for UI states:

```tsx
colors.semantic.success.DEFAULT // #388E3C
colors.semantic.warning.DEFAULT // #F57C00
colors.semantic.error.DEFAULT   // #C62828
colors.semantic.info.DEFAULT    // #0277BD
```

## Typography

### Font Families

```tsx
typography.fonts.display // Cormorant (serif)
typography.fonts.body    // Inter (sans-serif)
typography.fonts.sf      // SF Pro (system)
typography.fonts.mono    // Monospace
```

### Type Scale

Apple-inspired modular scale:

| Size | Value | Usage |
|------|-------|-------|
| `xs` | 12px | Labels, metadata |
| `sm` | 14px | Body small, captions |
| `base` | 16px | Body text |
| `lg` | 18px | Subheadings |
| `xl` | 20px | Headings |
| `2xl` | 24px | Section titles |
| `3xl` | 30px | Page titles |
| `4xl` | 36px | Hero headings |
| `5xl` | 48px | Display large |
| `6xl` | 64px | Hero display |

### Font Weights

```tsx
typography.weights.light     // 300
typography.weights.regular   // 400
typography.weights.medium    // 500
typography.weights.semibold  // 600
typography.weights.bold      // 700
```

```tsx

spacing[4]   // 16px

spacing[8]   // 32px
```

```tsx
spacing.xs   // 8px
spacing.sm   // 12px
spacing.md   // 16px
```

## Usage Example

```tsx
import { colors, typography, spacing } from '@/lib/design-tokens';

function MyComponent() {
  return (
    <div
      style={{
        backgroundColor: colors.gray[50],
        padding: spacing.lg,
        borderRadius: radius.xl,
        boxShadow: shadows.md,
      }}
    >
      <h2
        style={{
          fontSize: typography.sizes['2xl'][0],
          fontWeight: typography.weights.semibold,
          color: colors.brand.navy,
        }}
      >
        Premium Design
      </h2>
    </div>
  );
}
```

## Helper Functions

### Get Color by Path

```tsx
import { getColor } from '@/lib/design-tokens';

const navyColor = getColor('brand.navy'); // #243c74
```

### Responsive Font Sizes

```tsx
import { getResponsiveFontSize } from '@/lib/design-tokens';

const size = getResponsiveFontSize('2xl', '4xl');
// Returns: clamp(1.5rem, 2vw + 1rem, 2.25rem)
```

## Best Practices

<Callout type="info">

**Always use design tokens** instead of hard-coded values. This ensures:
- Visual consistency across the application
- Easy theme updates and maintenance
- Accessibility compliance
- Brand coherence

</Callout>
